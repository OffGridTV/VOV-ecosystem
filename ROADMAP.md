# VOV Ecosystem Roadmap

## Phase 1 — Repository Foundation
- Define canonical repo structure
  - `/assets`
  - `/components`
  - `/websites`
  - `/constitution`
  - `/investigations`
- Establish shared component architecture
- Add root-level documentation: `README.md`, `ARCHITECTURE.md`, `AI_CONTEXT.md`
- Create a minimal site layout for one proof-of-concept website
- Validate GitHub Pages compatibility

## Phase 2 — VOV Verified
- Complete `VOV Verified` homepage design and content
- Implement shared site components in `websites/vovverified`
- Verify responsive styling and accessibility basics
- Document content structure and key pages
- Confirm the site is deployable on GitHub Pages

## Phase 3 — VOV Standard
- Create `websites/vovstandard` with shared layout and branding
- Reuse root-level components for header, hero, CTA, footer, navigation
- Add content explaining the VOV Standard methodology
- Ensure visual consistency with VOV Verified
- Validate the second site in the shared architecture

## Phase 4 — Off Grid TV
- Create `websites/offgridtv` with shared foundation
- Define Off Grid TV messaging and navigation flow
- Reuse components and shared assets
- Ensure each site retains its own identity while staying consistent
- Verify multi-site maintenance workflow

## Phase 5 — Investigation Database
- Design a schema for investigation records and metadata
- Create a shared investigation listing page or section
- Add filtering/search basics for investigation content
- Establish content standards for evidence, sources, and conclusions
- Document the investigation content model and publishing process

## Phase 6 — Public Launch
- Final review of site performance and mobile behavior
- Complete copy editing and legal/privacy checks
- Prepare a launch checklist for GitHub Pages deployment
- Publish all three sites and confirm live URLs
- Document maintenance process and future expansion plans
